## What Went Well

1. **Interface-based testing** — DrugClient interface enabled full handler testing without upstream dependency. Mock variants (simple + call-counting) covered both happy and fallback error paths cleanly.

2. **TDD discipline** — RED-GREEN-REFACTOR cycle caught the url.QueryEscape security issue during verify, not in production. Tests drove the fallback NDC normalization logic naturally.

3. **Spec-driven approach** — Writing specs before code clarified the NDC format matrix (5-4, 4-4, 5-3) upfront. No ambiguity during implementation.

4. **CI pipeline iteration** — Despite 3 fix commits (gitignore, Go version, covdata), the pipeline stabilized quickly. Beta images now auto-publish on push to main.

5. **Coverage quality** — 97.1% overall with handler/middleware/ndc all at 100%. Only client.go at 91.3% due to one hard-to-test error path.

## What Didn't Go Well

1. **`.gitignore` pattern `server`** matched `cmd/server/` directory — caused CI failure. Root-only match (`/server`) was the fix. Lesson: always use anchored patterns for build artifacts.

2. **Go version chain reaction** — go.mod required 1.25.5, but Dockerfile and CI used 1.24. This cascaded into a `covdata` tool error in CI. Three commits to fix what should have been caught pre-push.

3. **swaggo generates Swagger 2.0, not OpenAPI 3.0** — The spec said "OpenAPI 3.0" but swaggo produces Swagger 2.0. Functionally fine but spec was inaccurate. Tests adapted with `HasSuffix` matching for definition names.

4. **No plan documents created before implementation** — ndc-lookup plan was retroactive. Docker and OpenAPI plans were skipped entirely. At alpha maturity this was acceptable but should tighten for M2.

## Learnings to Promote

### To User Library (~/.claude/add/library.json)

1. **Go .gitignore anchoring** — Always use `/binary-name` not `binary-name` when ignoring build artifacts, to prevent matching source directories with the same name. (Scope: workstation, stack: Go)

2. **Go version alignment** — Keep go.mod, Dockerfile, and CI workflow Go versions in sync. A mismatch causes cascading build failures. (Scope: workstation, stack: Go)

3. **swaggo produces Swagger 2.0** — Despite the ecosystem calling it "OpenAPI", swaggo/swag generates Swagger 2.0 specs. Definition names use full module paths as prefixes. Use `strings.HasSuffix` in tests. (Scope: workstation, stack: Go)
