# M10.5 — Landing Page

## Goal

Create a public landing page for drug-gate that explains what the service does, how to self-host it, and links to the GitHub repository. Serves as the front door for developers discovering the project.

## Appetite

2-3 days

## Success Criteria

- [ ] Landing page accessible at the root URL (or dedicated subdomain)
- [ ] Clearly explains what drug-gate does (drug information gateway, NDC lookup, interactions, etc.)
- [ ] Links to GitHub repo (github.com/finish06/drug-gate) for self-hosting
- [ ] Shows quick-start instructions (docker-compose up)
- [ ] Links to Swagger UI for API exploration
- [ ] Links to live staging instance for demo
- [ ] Responsive design (mobile + desktop)
- [ ] Deploys alongside the existing service without disrupting API routes

## Features

| Feature | Description | Position |
|---------|-------------|----------|
| Hero section | Project name, tagline, what it does | SHAPED |
| API overview | Key endpoints with example responses | SHAPED |
| Self-host guide | Docker Compose quick-start, env vars, prerequisites | SHAPED |
| Links | GitHub repo, Swagger UI, staging demo | SHAPED |
| Tech stack | Go, Chi, Redis, Prometheus — what powers it | SHAPED |

## Technical Considerations

- **Hosting approach:** Static HTML served by the drug-gate binary itself (e.g., at `/` or a separate nginx container), or a standalone static site on a subdomain
- **No framework required:** A single `index.html` with inline CSS is sufficient — no React/Vue/etc. needed
- **Must not conflict** with existing `/v1/*`, `/admin/*`, `/swagger/*`, `/health`, `/metrics` routes
- **Design:** Follow project branding from `.add/config.json` (ocean palette: #0891b2)

## Dependencies

- None — can be done independently of M10 or any other milestone

## Risks

- Route conflicts with existing API if served from same binary (mitigate: serve only at `/` with exact match, not prefix)
